---
title: "Quickstart: minify JavaScript with UglifyJS"
description: "Minify a JavaScript file from the CLI or Node.js API in minutes. Covers basic usage, compress and mangle flags, and error handling."
---

UglifyJS can shrink a JavaScript file in a single command. This guide walks you through installing the tool, running the CLI against a real file, enabling the full compress-and-mangle pipeline, and calling the same functionality from Node.js code. Each step builds on the last — you can stop once you have what you need.

<Steps>
  <Step title="Install UglifyJS">
    Install UglifyJS globally so the `uglifyjs` command is available in your terminal:

    ```bash
    npm install uglify-js -g
    ```

    Verify the install:

    ```bash
    uglifyjs --version
    ```
  </Step>

  <Step title="Minify a file from the CLI">
    Pass an input file with `-o` to write the minified output to a new file:

    ```bash
    uglifyjs input.js -o output.min.js
    ```

    Without `-o`, UglifyJS writes to stdout, which is useful for piping into other tools. If you do not specify an input file, UglifyJS reads from stdin.
  </Step>

  <Step title="Enable compression and mangling">
    Add `-c` (compress) and `-m` (mangle) to apply the full optimisation pipeline. This is the recommended invocation for production builds:

    ```bash
    uglifyjs input.js -c -m -o output.min.js
    ```

    Given this input:

    ```javascript
    function add(first, second) { return first + second; }
    console.log(add(1 + 2, 3 + 4));
    ```

    UglifyJS produces:

    ```javascript
    console.log(add(3,7));function add(n,d){return n+d}
    ```

    `-c` performs dead code removal, constant folding, and dozens of other transforms. `-m` renames local identifiers to short names (`first` → `n`, `second` → `d`). Both flags accept comma-separated options for finer control, e.g. `-c passes=2 -m reserved=['$']`.
  </Step>

  <Step title="Use the Node.js API">
    For programmatic use — such as inside a build script or bundler plugin — require `uglify-js` and call `minify()`:

    ```javascript
    var UglifyJS = require("uglify-js");

    var code = "function add(first, second) { return first + second; }";
    var result = UglifyJS.minify(code);

    console.log(result.error); // runtime error, or `undefined` if no error
    console.log(result.code);  // minified output: function add(n,d){return n+d}
    ```

    `minify()` enables `compress` and `mangle` by default. It never throws — check `result.error` before using `result.code`:

    ```javascript
    var result = UglifyJS.minify(code);
    if (result.error) throw result.error;

    // safe to use result.code here
    console.log(result.code);
    ```

    You can minify multiple files in one call by passing an object whose keys are file names and whose values are source strings. UglifyJS parses them in the same global scope, so cross-file references resolve correctly:

    ```javascript
    var result = UglifyJS.minify({
        "file1.js": "function add(first, second) { return first + second; }",
        "file2.js": "console.log(add(1 + 2, 3 + 4));"
    });
    console.log(result.code);
    // function add(d,n){return d+n}console.log(add(3,7));
    ```
  </Step>
</Steps>
